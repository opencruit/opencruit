# Ideas

Генератор идей по развитию текущего проекта. Формат — короткие абзацы по сути с тегами. Журнальный стиль: накапливаются со временем, потом можно пересмотреть.

## Хранение

```
.claude/ideas.md              # все идеи в одном файле
```

## Вызов

- `/ideas` — сгенерировать новые идеи по текущему состоянию проекта
- `/ideas <тема>` — сгенерировать идеи по конкретной теме (парсеры, UI, монетизация и тд)

## Поведение

### 1) Изучить контекст

Прочитать текущее состояние проекта:

- `CLAUDE.md` — структура, конвенции
- `docs/CONTEXT.md` — архитектура, подходы
- `docs/architecture.md` — верхнеуровневая архитектура
- `VISION.md` — продуктовое видение
- Текущие пакеты и их состояние

### 2) Проверить существующие идеи

Прочитать `.claude/ideas.md` (если существует). Это ОБЯЗАТЕЛЬНЫЙ шаг — нельзя пропускать.

### 3) Сгенерировать новые идеи

- 3-7 идей за вызов
- Каждая идея — 2-4 предложения максимум
- По сути, без воды
- С тегами в конце

### 4) Проверка дубликатов

Перед записью каждой идеи — проверить что она НЕ повторяет существующую:

- Не та же идея другими словами
- Не частный случай уже записанной идеи
- Не очевидное следствие уже записанной

Если идея дублирует — НЕ записывать, пропустить молча.

### 5) Записать

Добавить новые идеи в `.claude/ideas.md` с датой.

## Формат файла

```markdown
# Project Ideas

## 2026-02-23

Парсеры можно тестировать в CI через snapshot-тесты — сохранять эталонный output и сравнивать при каждом PR. Ловит регрессии без обращения к реальным API.
`#parsers #testing #ci`

Дедупликация может работать как отдельный микросервис с REST API — тогда её можно переиспользовать за пределами ingestion pipeline, например для пользовательских импортов.
`#architecture #deduplication`

## 2026-02-25

...
```

## Правила

- Идеи должны быть конкретными и реализуемыми, не абстрактными пожеланиями
- Не повторять то что уже записано — это главное правило
- Не предлагать то что уже реализовано в проекте
- Теги из набора: `#parsers`, `#sdk`, `#architecture`, `#ui`, `#search`, `#ai`, `#devex`, `#testing`, `#ci`, `#infra`, `#product`, `#monetization`, `#community`, `#deduplication`, `#events`
- Набор тегов не фиксированный — создавай новые по необходимости
- Файл `.claude/ideas.md` должен быть в `.gitignore`
