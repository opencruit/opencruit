# OPENCRUIT — Vision & Product Direction

**opencruit.me | Open-Source Job Aggregator Platform**

---

## 1. О проекте

**Проблема.** Рынок поиска работы разбит на десятки платных сервисов, каждый из которых берёт деньги за базовые функции: агрегацию вакансий, оценку резюме, трекинг откликов. Ни один не является open-source, не даёт контроля над данными и не позволяет self-hosting. Кандидаты платят $20–50/мес за сервисы сомнительной ценности, а их данные остаются у третьих сторон.

**Решение.** OpenCruit — полностью open-source платформа для поиска работы с AI-функциями, которую каждый может развернуть у себя бесплатно. Весь код открыт, включая AI-фичи. Монетизация через managed cloud hosting.

**Кому это.** Candidate-first: в первую очередь кандидатам (поиск и отбор вакансий, трекинг откликов, подготовка резюме под роль). При этом архитектура и продукт допускают расширение на рекрутеров/команды и hiring-процессы (например, lightweight ATS) на следующих этапах.

**Одной строкой.** _"Plausible Analytics, но для поиска работы" — open-source альтернатива платным джоб-бордам и трекерам откликов с AI-powered матчингом._

## 3. Бизнес-модель

**Полностью open-source (модель Plausible/Umami).** Весь продуктовый код открыт под permissive лицензией, включая AI-фичи. Пользователь клонирует репо, подставляет свои API-ключи, получает полный функционал бесплатно.

**Монетизация: managed cloud.** Платная версия — не закрытые фичи, а удобство: хостинг, инфраструктура, SLA, поддержка. Биллинг/Stripe живёт в отдельном инфраструктурном слое, не является частью продуктового кода.

- **Self-hosted (бесплатно):** пользователь сам хостит, сам платит за OpenAI/Anthropic API ключи, сам управляет инфраструктурой
- **Cloud (подписка):** мы хостим, мы платим за AI-токены, пользователь платит нам за удобство

**Почему не open-core:** проект называется OpenCruit — закрытые premium-фичи противоречили бы названию. Полная открытость = доверие = adoption = конверсия в cloud.

## 4. Ключевые фичи

### 4.1 Агрегация вакансий

- Плагинная система парсеров (каждый источник = отдельный плагин)
- Сбор данных с публичных страниц без авторизации (без обхода логина/ограничений; см. правовую секцию)
- Поддержка API там, где доступны
- Community-driven: разработчики пишут парсеры для новых сайтов
- Дедупликация вакансий из разных источников

### 4.2 AI-оценка резюме

- Анализ резюме по загруженному PDF/DOCX
- Скоринг соответствия конкретной вакансии
- Рекомендации по улучшению для прохождения ATS-фильтров
- Поддержка любого LLM провайдера (OpenAI, Anthropic, Ollama для self-hosted)

### 4.3 Создание резюме

- AI-генерация резюме под конкретную вакансию
- Шаблоны в разных форматах (PDF, DOCX)
- Кастомизация стиля и содержания

### 4.4 Трекер откликов (Job search CRM)

- Трекинг статусов откликов (applied, interview, offer, rejected)
- Канбан-доска для управления процессом поиска
- Напоминания и follow-up
- Аналитика: конверсия откликов, средние сроки, эффективность источников

### 4.5 AI-матчинг

- Автоматический подбор вакансий на основе профиля и резюме
- Scoring релевантности каждой вакансии
- Персональные рекомендации на основе истории откликов

## 6. Правовая база парсинга

Агрегация построена на сборе публично доступных страниц вакансий (без логина) и не предполагает обход авторизации. В США часто ссылаются на hiQ Labs v. LinkedIn (Ninth Circuit, 2019/2022) и Van Buren v. United States (Supreme Court, 2021) в контексте трактовки CFAA. При этом практика зависит от юрисдикции, условий сайтов (ToS) и конкретной реализации. Это не юридическая консультация.

**Принципы комплаенса (на уровне продукта):**

- Не использовать фейковые аккаунты и не обходить авторизацию/антибот
- Парсить только публичные страницы вакансий и сохранять ссылки на первоисточник
- Rate limiting, retries/backoff, и уважение к инфраструктуре источника
- Takedown-процесс: удалить/заблокировать источник по запросу при конфликте

## 7. Конкурентный ландшафт

**Платные SaaS (цель для disruption).** Десятки платных сервисов оценки резюме ($20–50/мес), ATS-трекеров, AI cover letter генераторов. Нулевой switching cost, нулевое доверие, высокий churn. OpenCruit убивает категорию за счёт бесплатности, открытости кода и community-driven развития.

**Open-source (пустая ниша):**

- Geezap (Laravel/PHP, 113 stars) — ближайший аналог, другой стек
- JoBoom (1 контрибьютор) — pet-project
- JobApis (PHP) — заброшен с 2017

Серьёзного open-source агрегатора с AI-фичами на современном веб-стеке пока не существует.

**Референсные модели:**

- **Plausible** — full OSS + paid cloud → прямая аналогия бизнес-модели
- **Cal.com** — full OSS + paid cloud → 10K+ stars, cloud revenue
- **Hirify.me** — closed SaaS → 60K вакансий/мес, 1200 платных юзеров = proof of demand

## 8. Стратегия запуска

Open-source — не только модель, но и маркетинговый канал. Для неизвестного разработчика единственный способ получить organic reach без бюджета.

**Фазы:**

1. **MVP + GitHub launch:** базовый агрегатор с 3–5 парсерами, AI-оценка резюме, минимальный ATS
2. **Community building:** Show HN, Reddit, Twitter/X
3. **Stars + adoption:** цель 1–5K stars, первые self-hosters, фидбек, контрибьюторы
4. **Cloud launch:** opencruit.me cloud hosting, подписка, конверсия 1–3%
5. **Масштабирование:** расширение парсеров, B2B-фичи, интеграции

**Каналы:** GitHub, Hacker News (Show HN), Reddit (r/opensource, r/webdev, r/cscareerquestions, r/nextjs), Twitter/X, Dev.to / Hashnode
